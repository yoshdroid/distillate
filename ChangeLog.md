# ChangeLog

## ver0.1

初版の最小実装を追加した。

### 実装意図

`original/distillate.py` は Pyxel を用いた水流シミュレーションの試作として意図が読み取れる一方、グローバル変数への依存、ロジックと描画の密結合、リスト走査中の削除、境界チェック不足、未完成の横流れ仕様といった問題を抱えていた。

そのため `ver0.1` では、元コードの見た目や雰囲気を保つことよりも、今後 AI と人間が安全に拡張しやすい土台を作ることを優先した。特に以下を重視している。

- Pyxel 依存部分とシミュレーションロジックを分離すること
- グリッド、ステージ、ブロック、水の責務を明確にすること
- 1 フレームの更新順序を固定し、バグの温床になりやすい副作用を減らすこと
- 最小実装でも後からステージ追加、ルール追加、テスト追加がしやすい構成にすること

### 採用した構成

- `main.py`
  - 起動用エントリポイント
- `distillate/config.py`
  - 定数、色、初期ステージ定義
- `distillate/stage.py`
  - タイル種別とステージ参照 API
- `distillate/models.py`
  - `WaterParticle`, `TempBlock` のデータモデル
- `distillate/simulation.py`
  - 水生成、排水、移動、ブロック寿命管理
- `distillate/input.py`
  - Bresenham によるドラッグ補間
- `distillate/renderer.py`
  - Pyxel 描画
- `distillate/app.py`
  - 入力、更新、描画の統合

### 初版で確定した最小仕様

- 31x23 グリッドの固定ステージをロードする
- 水源から水を継続生成する
- 水は下方向を優先し、塞がれていれば左右へ流れる
- 左クリックとドラッグで一時ブロックを配置できる
- 一時ブロックは寿命で消える
- 排水口の上下左右に隣接した水は削除される
- `W` キーで全水削除と再生成クールダウンを行う
- `Q` キーで終了する
- デバッグ情報を画面表示する

### 元コードからの主な改善点

- 水とブロックを座標辞書で管理し、重複チェックと削除を簡潔にした
- 境界判定を `Stage` に集約し、外部ステージでも壊れにくくした
- リストを走査しながら削除する危険な実装を排除した
- 背景、ブロック、水、デバッグ表示の描画順を整理した
- 横流れ仕様を「下優先、次に左右、競合は先着」で明文化した

### このバージョンで意図的に入れていないもの

- 勝利条件、敗北条件、スコア
- 複数ステージ管理
- ファイル外部ロード
- サウンド
- 厳密な流体シミュレーション

### 今後の拡張候補

- ステージデータの外部ファイル化
- クリア条件や目標導線の追加
- デバッグ表示の切り替え
- 水挙動の高度化
- テストコードの追加
