## 1: 青から赤への変化を調整する
- ステージパラメータとして、初期水ブロックが最初から赤で生成される確率を新設する
    - 適切なパラメタ名を決めてほしい
    - デフォルトは 0.05
    - この値が 0.0のとき これまでと同様に最初は必ず 青い水ブロックとして生成される
    - 各ステージデータに記載されていればその値で上書きする
- 青い水ブロックは初期生成時に MAX_STRESSの 0.0から 0.1倍の範囲のランダムな整数値を stressに設定する
    - 青から赤へ変化するタイミングをそろえすぎないことが目的
- ここまでの変更を整理、記載し git commitしてください
    - 次の実装に進む前に、私も動作を確認したい

## 2: 効果音を鳴らせるよう準備する
- sound.pyとして実装を切り出すことが可能か方針を最初に検討する
- pyxelのサウンド機能のうち 0チャンネルを効果音に割り当てる
    - 効果音を鳴らすタイミングは追加できるよう拡張性をもたせる
    - 効果音種類と優先度の 定義は下記。優先度は、同時発音となった時 数字のより小さいものを再生する。
    - 効果音デフォルトテンポ T240
        - 優先度 0(最高): システム音
            - qキー入力による画面遷移
            - wキー入力によるリスタート
        - 優先度 1(高): ユーザ操作に対する効果音
            - マウスクリック入力
                - 壁ブロックの設置
                    - ノイズ音設定で スネアのような音を鳴らす
                - （将来実装予定機能）壁ブロックの破壊
                    - ノイズ音設定で クラッシュシンバルのような音を鳴らす
        - 優先度 2(中): ゴール用の排水ブロックで 赤い水の排水時
            - パルス音設定で 短く o2 c16c#16 を鳴らす
        - 優先度 3(低): ゴール用の排水ブロックで 青い水の排水時
            - パルス音設定で 短く o4 d16 を鳴らす
        - 優先度 4(最低): 通常の排水ブロックでの排水時
            - パルス音設定で 短く o3 c16 を鳴らす
- 1から 3チャンネルを使って 効果音とは排他的に BGMを流せるようにする
    - 複数曲を切り替えられる
    - 将来 私が MMLをつかって BGMを書けるようにスケルトンコードを準備してほしい
    - 再生と停止を実行指示できるようにしてほしい
        - 主に、ステージ開始で再生、ステージクリアや終了で停止
    - フェイドインとフェイドアウトが実現可能か調査検討し、可否を教えて。できたら実装し、できなくてもそのまま次へ進む
- ここまでの仕様追加を整理、記載し git commitしてください
    - 次の実装に進む前に、私も動作と実装内容を確認したい

## 3: 壁ブロックを破壊可能にする
- 壁ブロックを左クリックしたとき、効果音とともに 破壊(マップデータを 0に変更)する
- マウスドラッグでユーザ壁を連続配置中に 既存の壁を通過した場合、破壊しない
- データ上 1の壁の破壊について
    - グリッドの最上段と最下段は連続するように取り扱う
        - たとえば: 最下グリッドにて落下判定をするとき 最上グリッドの空きを確認し、空いていたら移動可能となる
    - 同様に グリッドの左端と右端とは連続するように取り扱う
    - wキーでのリスタート時には初期配置に復元する
- 左クリックして設置されたユーザ壁の破壊について
    - wキーでのリスタート時にはすべて削除する
        - 破壊の効果音は鳴らさない
- ここまでの仕様変更を整理、記載し、 ver0.7として git commitしてください
